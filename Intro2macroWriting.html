Why macros?

  • Automation: do the same thing many times over
  • Reproducibility: do exactly the same thing every time, show the world what you've done
  • Sharing: let someone else do exactly the same analysis as you
  • Extension: do things Fiji cannot do by default

"My first proper macro"

Objective: apply a simple analysis pipeline over a set of images

  • Data: files
  • Save these images to your desktop in a folder called "Input"
  • Make a new folder called "Output"
  • Find the ImageJ macro template "Process_Folder"

Hints

  • Use each macro to make the next one.
  • Remember the macro recorder.

Task 1:

  1. Open every image and print the Image name to the log.

\newline (Hint: What do the variables input, output and file mean?)

Task 2:

  1. Open every image, print how many slices there are in the image and then close the image.
  2. i.e. you shouldn't have all of the image open at the end

Task 3:

  1. Open every image, print how many slices there are in the image...
  2. do a Z projection between slices 3 and 7...
  3. then save this as a seperate image (with a suitable filename).
  4. Close all the images.

Task 4:

  1. Open every image, print how many slices there are in the image then ...
  2. do a Z projection between slices 3 and 7,...
  3. subtract the background fluorescence...
  4. then save this as a seperate image (with a suitable filename).
  5. Close all the images.

Task 5:

  1. Open every image, print how many slices there are in the image then ...
  2. do a Z projection between slices 3 and 7,...
  3. measure the "mean gray value"...
  4. subtract the background fluorescence...
  5. then save this as a seperate image (with a suitable filename).
  6. Close all the images
  7. Save the results table as a .csv file.
